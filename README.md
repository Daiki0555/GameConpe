# **6.基本的な動作**
>## **オートランゲームのため常時走り続けています。**
>## **そして重力に関しては、その時の重力の向きに対する高さの座標を一定数値にすることで地面にいるみたいに表現しています。**
>## **そのためジャンプしたときにのみ、重力が効くようにしています。**
![基本説明１](https://user-images.githubusercontent.com/122655581/218236248-0d446567-9ea8-494f-99bd-087e933263ea.png)

<br>

# **7.技術紹介**
># **a.重力の向き変更**
>## **重力の向きはStateで重力の向きを管理しています。**
>## **そのため重力の向きを変えるときに、向きに対する高さの座標を少し上げることによって、無理やり重力を発生させています。**
> ![重力説明](https://user-images.githubusercontent.com/122655581/218256413-59e624b1-5660-4913-b0d3-f4c92f733e82.png)
>## **しかしこれだけでは、一つ問題が発生してしまいます。**
>## **それは、重力を変更したときにジャンプをしていると、そのまま壁に移動するのではなく斜め方向に移動し続けてしまいます。**
>## **その問題を解決したのが次のプログラムです。**
> ![image](https://user-images.githubusercontent.com/122655581/218272314-fb0b409d-6339-49b7-9a97-219a542f9855.png)
>## **先のif文の否定のプログラムで、重力を絶対値によりジャンプをしている判定にし重力を発生させ、**
>## **元の高さの座標を0にすることでその問題を解決しています。**

># **b.ステージの生成**
>## **このゲームは長いステージを作るのでは無く、短いステージを一定の距離に行った時に生成をするようにしています。**
>## **それにより世界がでかくなりすぎないようにして、処理を軽くしています。**
>## **そして、ステージの生成方法ですが、それは透明なブロックをプレイヤーの座標にし、そしてその透明なブロックが生成の判定をする足場と重なった時に判定をとるようにしています。**
> ![ステージの追加](https://user-images.githubusercontent.com/122655581/218308105-ae421054-6ebf-47cf-be4a-18de397481ca.png)
> ## **しかし、重なっている間ずっと生成されてしまうので足場の判定を削除するようにしています。**
> ## **ただ、同じステージを作成するのではなくランダムでステージを作成するようにしました。**

># **c.落下判定**
>## **落下判定はさっきのステージ生成と同じように、プレイヤーの座標にある透明なブロックが落下判定用の透明なブロックとあった時に判定をとっています。**
>## **しかし、判定を細かくしないと、小さい穴などが判定を取れなかったためこのようにしました。**
>![判定線](https://user-images.githubusercontent.com/122655581/218312433-a27ec8d9-fdec-4860-b379-a33f5d81d2da.png)
>## **このように透明判定のブロックを小さくし敷き詰めるようにし判定をとるようにしました。**
>## **さらにプレイヤーの位置にある透明のブロックを細くすることにより違和感がない用に落下させる事が出来ました。**
>## **そしてこれが落下判定のプログラムです。**
> ![落下判定](https://user-images.githubusercontent.com/122655581/218313392-b41efa2e-7431-41db-b28a-de0b394298bd.png)
